**Консьюмер**(Consumer) — это клиентское приложение или программа, которая подписывается на один или несколько топиков (каналов для сообщений) и извлекает (читает) сообщения из них, обрабатывая их в реальном времени. График взаимодействия [[Продьюсер]] и консьюмеров в Kafka работает по принципу pull-модели (сам запрашивает данные), отслеживает, какие сообщения прочитал (через оффсеты), и может работать в группах для масштабирования и отказоустойчивости, где каждая [[Партиции]] обрабатывается одним консьюмером.


Основные функции консьюмера:

- **Подписка:**
    
    Сообщает брокерам Kafka, из каких топиков он хочет получать сообщения. 

- **Извлечение:**
    
    Запрашивает и получает новые сообщения из партиций топиков. 
    
- **Обработка:**
    
    Выполняет необходимую логику над полученными данными (например, сохраняет в базу данных, анализирует). 
    
- **Отслеживание:**
    
    Запоминает позицию (оффсет) последнего прочитанного сообщения для возобновления чтения с нужного места.


Доп:
**Offset:**
- Уникальный номер сообщения внутри партиции. Консьюмер сохраняет этот номер, чтобы знать, с чего начать чтение.
